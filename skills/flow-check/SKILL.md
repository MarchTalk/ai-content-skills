# flow-check · 完播体检

## 核心判断

观众中途走掉，大多不怪主题，怪某一个具体的点：两段之间断了一步逻辑、某一段在原地绕圈、某句话根本念不出口。这些断点找得到，也改得掉。

质检单位有两级：**句子层和叙事层**。句子全部通顺，片子照样可能不成立——时间线乱了、牌提前掀了、情绪原地踏步，都是句子层查不出来的病。所以先扫句子层三项；带画面的脚本，必须再过叙事层五项。

## 什么时候调用

- 逐字稿 / 图文长文成稿后、交付前。
- 用户说「帮我看看顺不顺」「哪里会让人划走」「怕完播低」。
- vlog / 分镜脚本写完（叙事层五项强制，不许跳）。

## 不要做什么

- 不要改观点、案例、数据。这道体检只管怎么说，不管说什么。
- 不要把用户的话改出 AI 味。给的修补句必须能直接张嘴念出来，保持用户自己的语气。
- 稿子整体顺就说顺，不要硬找茬凑报告。
- 不要在自己给的修补句里犯排比、报幕、翻转句式这些病——医生开的方子里不能带病。

## 工作流程

### 第一层：接缝扫描（段与段之间）

逐个接缝问同一个问题：**上一段的最后一句，能不能自然带出下一段的第一句？**

- 带不出来 = 中间漏了一步逻辑。补上那一步，或者调整段落顺序。
- 禁止用连接词硬粘。「所以 / 因此 / 而 / 于是 / 值得注意的是」堵在缝上，缝还是开的，读者照样觉得奇怪。
- 禁止报幕句过渡。「接下来说说 X」「给你算笔账」「听我掰扯掰扯」——这类句子零信息，是写稿人在主持节目，真人聊天不报幕。检查法：把每个过渡句单独拎出来问「这句有没有信息量」，没有就删掉，让两段直接相接；接不上，说明结构顺序错了，调顺序，不是加报幕。

### 第二层：密度扫描（段落内部）

- **重复**：同一个意思换了几种说法反复讲，留最有力的一种，其余删掉。
- **绕圈**：五句话讲一句话就能说清的事，压缩。
- **正确的废话**：把这段删掉，观众有没有损失？没有就删。
- **结构病专项**（AI 稿的重灾区）：机制、原因、并列要点这些自带逻辑结构的段落，最容易把逻辑结构直接写成句式结构——工整对仗、三连排比、同构问句串（「A 不 A？B 不 B？会不会 C？」）。检查法：把连续几句竖排看一眼，像一个模子刻出来的就拆——并列的几点装进不同句型里，一句陈述、一句画面、一句反问。逻辑上的「三条」，不许变成句式上的「三连」。

### 第三层：口感扫描（句子）

核心动作是**默读**：把每一段当成要张嘴讲出来，读一遍，凡是嘴上根本不会这么说的句子，就地改。

- 倒装 → 正常说话语序，主语先出来（谁 + 怎么了 + 咋样）。
- 抽象名词当主语宾语 → 拆成人能做的动作、能指的东西。
- 一句塞太满 → 拆短。一句只装一个意思，真人说话会停、会重启。单句超过 30 字就要警惕。
- 术语后面紧跟一句大白话解释，不许裸奔。

判断标准只有一条：**读出来像不像一个真人在跟你聊天。不像，就没写完。**

### 第四层：叙事层五项（带画面的脚本强制，纯口播可跳过）

逐镜回答五个问题。答不上来的镜头就是断点——**调结构，不是润句子**：

1. **时间线**：镜头顺序是不是一个正常人做这件事的真实次序？（反例：拖着行李玩了一整天，最后才进酒店办入住。）
2. **道具动线**：每一镜「人在哪、关键道具在哪」答得上来吗？（行李不能中途凭空消失。）
3. **悬念翻牌**：观众到第 N 秒已经知道了什么？「揭晓」给的是不是新信息？同一个信息换句话再讲一遍，不算揭晓。
4. **情绪推进**：情绪是一条往前走的线，还是同一个点原地踏步？（连续三次「被风景震住」是同一拍。）
5. **声画对位**：旁白讲的是画面拍不到的东西（心理、因果、画外信息），还是在解说画面正在演的事？画面已经在演了旁白还在念，等于浪费一条声道。

### 输出格式

逐处引用原文 → 说清这里为什么会走人 → 给一句能直接念出来的改法。按两档标：

- **【必改】**：大概率在这里走人（逻辑断裂、假揭晓、整段废话）。
- **【可改】**：改了更顺，不改也能发。

最后一句话决断：必改几处，改完能不能发。
