# topic-strategy

## 什么时候调用

- 定位完成后，需要生成 10-30 条可执行选题。
- 账号主"不知道下一条拍什么"。
- 老账号数据掉了，需要换一批选题方向。
- 大节点 / 大事件 / 价格争议 / 行业风波，需要顺势选题。
- 个人 IP 要建一批深度内容、价值观稿、人物观察稿。

## 不要做什么

- 不要列"通用爆款选题表"（"开头怎么写""文案模板"），脱离具体的人 / 产品 / 阻力没价值。
- 不要把同行选题原样照搬，必须按自己账号的限制条件重新过滤。
- 不要凑数。一条没有商业承接的选题，不等于无效；先判断它是不是在做人设背书、价值包装、信任铺垫或同频筛选。只有既不服务成交、也不服务人设 / 信任 / 同频的选题才砍掉。
- 不要混淆"我想讲的"和"用户想看的"，但**也不要走另一个极端，把所有"我想讲的"都砍掉**——个人 IP 的深度内容本来就是从"我想讲的"开始。
- 不要把选题写成"如何 / 几种 / 三招"这种课程目录腔。
- 不要让所有选题都围绕变现。一个健康的选题池里，人设 / 价值观 / 思考类应该占 20-40%（个人 IP 更高，企业号略低）。
- 不要把"承接"字段理解成"必须引导私信 / 购买 / 到店"。承接可以是软植入、认知铺垫、信任背书、人设记忆点，也可以明确写"本条不承接"。

## 工作流程

第一步：先判断人性需求 + 战略意图

不要先从"我有什么素材"开始，先判断目标用户此刻需要什么：

| 人性需求 | 用户想要 | 常见内容意图 |
|---|---|---|
| 安全 / 确定 | 降低风险、少走弯路、知道下一步 | 信任、行动、工具 |
| 认同 / 归属 | 确认"不是我一个人这样"、找到同类 | 共鸣、族群定义、人设 |
| 成长 / 超越 | 看见路径、方法、可能性 | 向往、案例、方法 |

再选择内容形式：

| 形式 | 战略意图 | 用法 |
|---|---|---|
| 说观点 | 定义族群 | 用鲜明立场筛选同频人，凝聚核心粉丝 |
| 讲故事 | 制造共鸣 | 让目标用户在故事里看见自己，觉得"你懂我" |
| 晒案例 | 缩短路径 | 证明方法有效，展示转折、工具、过程和可复制步骤 |
| 教知识 | 交付杠杆 | 给一个立刻能用的工具，降低行动门槛 |
| 追热点 | 嫁接注意力 | 借热点切入，但必须回到自己的行业判断 |

关键规则：形式服务意图。不要为了发故事而讲故事，不要为了晒案例而晒结果，不要为了追热点而离开账号主线。

同时落到动态内容生态五类资产：

| 生态资产 | 作用 | 判断标准 |
|---|---|---|
| 立人设内容 | 信任资产 | 是否回答"你是谁、我为何信你" |
| 干货内容 | 价值资产 | 是否回答"你能给我什么" |
| 营销内容 | 转化资产 | 是否回答"我为何现在行动" |
| 共鸣内容 | 黏合剂 | 是否精准描绘目标用户共同的感受 / 状态 / 困惑 |
| 关系内容 | 催化剂 | 是否把单向输出变成互动、幕后、衍生或反馈 |

注意：这里的"营销内容"只指直接号召 / 叫卖那一格，正常占 5-10%。人设、干货、共鸣、关系也都在广义营销，只是营销的是这个 IP 的可信、可用、可亲、可近，不要把它们误判成"不服务商业"。

第二步：选题来源

按下面八个口子取材，每条选题要能对应到至少一个：

**商业向（服务成交）**

1. **用户处境**：目标用户当下在经历什么具体场景、什么具体动作、什么具体决定。
2. **行业矛盾**：行业里大家心知肚明但没人公开讲的事（差价、套路、潜规则、信息不对称）。
3. **成交阻力**：用户为什么犹豫不买（钱 / 风险 / 时间 / 信任 / 面子 / 决策成本）。
4. **价格争议**：定价是怎么算的、贵的贵在哪、便宜的为什么便宜、什么样的人值得花贵的钱。
5. **客户误解**：客户经常搞错的常识、被同行误导的判断、自己以为对其实错的决定。
6. **真实案例**：手里的客户故事、订单经过、售后情况、回头客来源。

**人设向（服务 IP 建立）**

7. **人物观察 / 价值观**：你观察到的人 / 事 / 行业 / 时代里，让你形成了哪些判断（不为成交，为让同频的人看见）。
8. **长期思考 / 自我观察**：你长期在想的、还没想透的、想和大家一起想的问题（不为成交，为建立思考的深度感）。

第三步：过滤

每条候选选题过六道闸：

- **能不能精准吸到目标客户 / 同频人**：成交型问能不能吸到付费客户；人设型问能不能吸到同频的人（同频的人未来可能成交，也可能只成为长期粉丝，都算数）。
- **能不能拍**：账号主的限制条件允许吗（能不能出镜 / 有没有素材 / 有没有现场 / 有没有这个思考深度）？
- **形式是否服务意图**：观点是否在定义族群？故事是否在制造共鸣？案例是否缩短路径？知识是否交付杠杆？热点是否嫁接回账号主线？
- **有没有用途**：成交型问商业落点；信任型问建立什么判断 / 信任；人设型问"这条立住什么人设印象 / 让什么人觉得被看见"。
- **有没有平台合规风险**：行业红线 / 平台限流词 / 敏感品类。
- **新不新**：是不是已经被同行写烂、用户已经审美疲劳的角度？

过不了六道闸的直接砍。

第四步：分类

先打"生态资产"，再打"策略类型"。

生态资产五类：

- **立人设内容**：信任资产，回答"你是谁 / 我为何信你"。
- **干货内容**：价值资产，回答"你能给我什么"。
- **营销内容**：转化资产，回答"我为何行动"，只占少数，不要泛化成所有内容。
- **共鸣内容**：黏合剂，精准描绘目标用户共同感受。
- **关系内容**：催化剂，互动 / 幕后 / 衍生 / 评论区反馈。

策略类型五类：

- **流量型**：让陌生人刷到、记住、关注。
- **信任型**：让看过的人觉得"这家懂行 / 不忽悠"。
- **成交型**：让信任过的人推一把就下单。
- **人设 / 价值观型**：让用户认识 IP 这个人、记住他的判断。
- **思考 / 观察型**：把一个困惑 / 观察讲清楚，让同频的人共鸣。

发布节奏建议（按账号性质）：

- **个人 IP（强人设方向）**：流量 30% / 信任 20% / 成交 10-20% / 人设·价值观·思考 30-40%。
- **企业号 IP（创始人专栏 + 产品内容）**：流量 30% / 信任 25% / 成交 25% / 人设·价值观·思考 20%。
- **企业号 IP（纯产品 / 门店 / 工厂）**：流量 40% / 信任 30% / 成交 30% / 人设 0-10%（创始人栏目除外）。

不是死数，按账号的实际目标调，但**纯成交内容不能超过一半**，纯人设内容也不能低于 10%（除非是纯产品号）。

动态内容生态的基础配比：价值干货 50-60% / 情感共鸣 20-30% / 人格信任 10-20% / 用户关系灵活穿插 / 转化营销 5-10%。编排节奏不要连续多条硬干货，也不要连续多条共鸣，优先用"干货 → 共鸣 → 人设/幕后 → 干货 → 互动"。

第五步：每条选题的字段

每条选题必须写齐这几项：

- 标题方向 / 一句话内核
- 生态资产（立人设 / 干货 / 营销 / 共鸣 / 关系）
- 类型（流量 / 信任 / 成交 / 人设 / 思考）
- 人性需求（安全/确定、认同/归属、成长/超越）
- 战略意图（定义族群 / 制造共鸣 / 缩短路径 / 交付杠杆 / 嫁接注意力）
- 内容形式（说观点 / 讲故事 / 晒案例 / 教知识 / 追热点）
- 目标用户（同频人 / 目标客户）
- 平台（主战场 + 可改的版本）
- 流量理由或共鸣理由（成交 / 信任 / 流量型写"用户为什么刷 / 搜"，人设 / 思考型写"什么人会看完后觉得被说中"）
- 承接 / 人设目的（成交型必填具体动作；信任型写软植入或信任目的；人设 / 观点 / 故事 / 思考型写"建立 ____ 的人设印象 / 价值包装"，也可以写"不挂承接"）
- 风险点（合规 / 同行打架 / 评论区翻车 / 人设和现有定位冲突）
- 来源（八个口子之一）

第六步：排序

- 先排"低风险 + 能解决当前账号最卡问题"的，不默认等于高商业承接。
- 成交短板严重时，优先排低风险成交型。
- 人设 / 信任短板严重时，优先排认知、观点、故事、信任型。
- 再排"中风险 + 高流量 + 能转信任"的（信任型）。
- 再排"能立人设"的（人设 / 价值观 / 思考型）——这类内容是底色，要均匀分布在发布节奏里，不要堆在一个时段。
- 最后排"纯流量但能立住人设印象"的（流量型）。
- 风险高、用途弱、复用度低的直接砍掉。

## 质检标准

- 每条选题字段写齐，没有"待定"。
- 每条选题都能说清楚：人性需求 → 战略意图 → 内容形式 → 行业场景。
- 每条选题都能落到动态内容生态五类之一：立人设 / 干货 / 营销 / 共鸣 / 关系。
- 至少一半选题来自账号主自己的真实素材（用户、订单、客户故事、个人观察、长期思考），不是凭空想。
- 选题里没有"如何 / 几种 / 三招"这种课程目录腔的标题方向。
- 选题池里同时覆盖商业向（流量 / 信任 / 成交）和人设向（人设 / 价值观 / 思考），按账号性质配比合理。
- 选题池里直接叫卖型营销内容没有被放大到过高比例；人设 / 干货 / 共鸣 / 关系没有被误判为"不营销"或"没承接"。
- 成交型有具体承接动作（评论引导 / 私信钩子 / 主页 / 朋友圈 / 到店）。
- 信任型可以软植入，也可以不挂硬承接，但要说明建立什么信任。
- 人设 / 观点 / 故事 / 思考型不挂承接时显式说明"建立什么人设印象 / 价值包装"，没有被误判为"承接缺失"。
- 排过序，不是堆完就交。
- 砍掉的选题有理由（哪条没过哪道闸）。
