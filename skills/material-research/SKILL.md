# material-research · 实料调研（防口水话 · 写稿前强制前置）

## 这个 skill 干什么

**每次写稿——不论给谁、写什么——动笔前都必须先跑这个 skill。** 它是所有写稿流程动笔前的强制前置闸门。

为什么：「太泛 / 口水话」的根是料太空——没有「只有查过才知道」的具体细节。去 AI 味 / 去排比只守下限（不难看），守不了上限（有信息密度、能被收藏转发）。爆款的料只能来自真实调研或第一手判断，凭通用先验硬写必然口水话。

一条底层判断贯穿全程：**深度 = 别人没有的料（第一手信息）+ 别人没有的看法（独到见解）。** 「查 / 不查」的分流就是从这条来的。

## 第一步：内容类型分流（每次都做）

| 类型 | 处理 |
|---|---|
| 干货 / 知识 / 科普 / 数据 / 测评 / 盘点 / 教程 / 平台玩法 / 行业现状 | **走 A：强制联网调研** |
| 观点 / 认知 / 人设 / 价值观 / 思考 / 情绪共鸣 / 个人故事 | **走 B：判定放行** |

拿不准就问一句：**这条的价值主要来自「我查到的事实」还是「我独有的判断」？** 前者走 A，后者走 B。混合型（既有判断又需事实佐证）→ 走 A，但只查支撑判断的关键事实，不喧宾夺主。

## 第二步 A：联网调研（facts 型）

用 WebSearch / WebFetch 真查（不许编、不许拿先验冒充）：
1. 这个题目下**真实的具体案例 / 数字 / 最新动态 / 平台规则 / 时间线 / 一手细节**——能写进稿子当血肉的硬料。
2. 同题**爆款 / 高赞是怎么讲的**（钩子、切入点、被收藏的理由）——找信息密度标杆，不是抄观点。
3. 涉及平台规则 / 数据 / 时效性强的，优先查最新（注意当天日期），旧数据标注时间。

**调研 ≠ 拼凑报告：**
- ✅ 好调研：具体的人 / 事 / 数 / 最新变化 → 「只有查过才知道」的细节。
- ❌ 坏拼凑：堆宏观报告数字 + 名人语录撑场面 = 穿西装的口水话；或摊成「什么是 X 的三个原因」科普流水账。
- 判据：**删掉这条料，稿子会不会塌一块？会 → 血肉，留；不会 → 装饰，删。**

产出「实料清单」3-8 条，每条带来源。**清单为空 = 没真查 = 不许往下写。**

## 第二步 B：判定放行（judgment 型）

不联网查外部料。**显式写一句放行声明**，说清这条为什么靠第一手判断而非外部资料——把「判断型」和「偷懒没查」彻底分开。

观点稿的力量来自独有判断和亲身得失，掺二手料会稀释。但写稿时仍可用 IP 本人的真实经历、客户案例当证据——那是第一手，不是外部调研。

## 输出格式

```
内容类型：____ → 走 A（查）/ 走 B（放行）

【A 型 · 实料清单】（3-8 条，带来源）
1. {具体案例 / 数字 / 动态} — 来源：{出处} — 用在：{稿子哪段}
2. ...
（清单为空则不许进写稿）

【B 型 · 放行声明】
本条是 ____ 型，价值来自 ____（第一手判断 / 亲身经历），无需外部二手料调研。
证据用：____（自己的真实经历 / 客户案例，如有）。
```

## 硬规矩

1. **每次写稿都跑，不论给谁写、写什么形态**（口播 / 图文 / 长稿 / 朋友圈都算）。
2. A 型清单为空 = 没真查 = 不许往下写；不许拿「我知道」冒充查过。
3. B 型必须显式放行声明，不许默默跳过（否则分不清「判断型」和「偷懒」）。
4. 调研只为信息密度和具体，不为撑场面；「删掉会塌」是唯一留料判据。
5. 跑完把结果交给调用它的写稿流程，写稿时逐条用上 / 或说明为什么不用。

## 什么时点调它

- 任何写稿流程在「选完角度、动笔前」调本 skill。
- 也可单独调用：手里有个选题，先查料再说。
- 写完的稿子交付前建议再过一道 `quality-gate`（最终质检）。

## 语言

中文。中英文之间加空格、数字与中文之间加空格。
