# quality-gate

## 什么时候调用

- 任何稿子在交付前必经一道。
- 各写稿流程出稿后。
- 用户问"这条能发吗"。
- 用户问"我自己写的这条有什么问题"。

## 不要做什么

- 不要做温和质检（"整体不错，建议在 X 方面优化"），要直接标问题。
- 不要用一套清单质检所有稿子，不同内容类型用不同清单。
- 不要因为一条人设 / 价值观 / 思考型内容"没有商业承接"就否决——那是定位特征，不是缺陷。
- 不要因为一条成交型内容"不够有人设"就否决——成交型的核心是动作转化，不是人设。
- 不要在质检里给"建议"，要给"修改方向 + 重写示例"。
- 不要在没有原稿的情况下质检（要看具体的句子才能定位问题）。

## 工作流程

第一步：先认定内容类型

读稿之前先和上一步对齐：这条稿子是流量 / 信任 / 成交 / 人设 / 价值观 / 思考 中的哪一类？

如果不清楚，回去找定位再质检。

第二步：通用质检（所有类型都要过）

不管什么类型，先过这九条：

1. **空话**：有没有"很重要 / 至关重要 / 关键的是 / 核心在于 / 我们要 / 大家都"这种没动作的句子？
2. **模板腔**：有没有"今天告诉你 / 三个秘密 / 你有没有发现 / 一个真相 / 干货分享"这种开头？
3. **AI 腔**：有没有"总而言之 / 综上所述 / 在这个充满 ____ 的时代 / 让我们一起 / 携手共创"？
4. **机械排比**：有没有连续 3 句以上"不是 ____ 而是 ____"或"是 ____ 是 ____ 也是 ____"？
5. **重复句式**：有没有连续 2 句以上用同一种句式开头？
6. **书面词**：有没有"赋能 / 势能 / 底层 / 闭环 / 降维 / 沉淀 / 链路 / 抓手 / 心智"塞进口播 / 朋友圈 / 短文本？
7. **目标用户错位**：这条内容的目标用户和稿子里隐含的"在跟谁说话"是不是一致？
8. **现场感**：有没有具体地点、具体时间、具体人、具体动作、具体话？至少 2 处。
9. **结尾**：有没有"以上就是今天的分享 / 觉得有用点赞关注 / 私信领取"这种低质引导？

任一条未过，标问题 + 给改写方向。

第三步：分类型质检

**流量型（让陌生人刷到 / 记住）**：

- 前 3 秒（口播）/ 封面（小红书）/ 第一句（图文）有没有钩子？
- 中段有没有钩子分布？
- 有没有让目标人群感到"被说中"的那个点？

**信任型（让用户觉得这家懂行）**：

- 有没有"行业里没人讲的具体一件事"？
- 是不是堆术语装专业 vs 用具体场景显出懂行？
- 评论区会不会有人来认领、追问、补充？

**成交型（让信任的用户下单）**：

- 是不是针对一个具体阻力（钱 / 风险 / 时间 / 信任 / 面子 / 决策成本）？
- 有没有具体承接动作（评论引导 / 私信钩子 / 主页 / 朋友圈 / 到店）？
- 承接动作是否可执行（"评论区固定回复 ____"这种程度具体）？
- 有没有"用户看完该做什么"的明确指引？
- **合规红线（内容含 价格 / 收费 / 召集 / 导私域 / 导微信 / 到店导流 时强制过闸，不过不放行）**：
  - **比对平台官方红线原文**（不是凭印象）：小红书去「学习中心」查笔记运营规则（不规范营销笔记会被限制流量曝光，另注意夸大宣传 / 低俗擦边 / 重复发布），抖音去「电商学习中心」查规则（重点：诱导第三方 / 脱离平台交易、严重违规营销、虚假宣传、价格违规）。规则会更新，以官方当前原文为准。
  - **导流尺度**：站内不直接放微信号 / 二维码 / "加我"等脱离平台话术（抖音明令打击；小红书会限流曝光）。承接只走平台允许的方式（私信钩子 / 主页 / 评论区固定回复 / 到店）。
  - **账号商业资质**：个人号未挂商业组件（抖音企业认证 / 本地生活、小红书专业号 + 店铺 / POI）时，价格招揽类内容默认不进公域发布排期，先走私信 / 置顶评论 / 到店物料，或等商业能力开通。
  - **结论要诚实**：分清"平台官方原文明令"和"行业通行认知"，下合规判断时标清楚哪条是白纸黑字、哪条是经验。

**人设 / 价值观 / 思考型（建立人设印象 / 让同频人共鸣）**：

- 有没有立住一个判断？这个判断换个人说不一样吗？
- 有没有具体的得失 / 代价 / 观察来支撑判断（不是抽象金句）？
- 哪种人会觉得"被说中"？
- **不挂商业承接** 是否有显式说明（"本条目的是 ____"）？没有显式说明不算缺陷，但写出来更稳。
- 有没有被错误地按"成交型"标准做？

第四步：标问题 + 改写

每个问题按下面三段写：

```
### 问题 N：____（一句话）

问题句：
> "____"（摘原稿）

为什么是问题：
____（指向具体规则）

改写方向：
- 改写示例 A：____
- 改写示例 B：____
```

至少给一个改写示例，不要只给"建议改成更有现场感的表达"这种空话。

第五步：通过判定

- **通过**：所有应过的清单都过。
- **改后通过**：1-3 个小问题，按改写方向改一下就行。
- **不通过**：≥ 4 个问题，或核心问题（目标用户错位 / 内容类型搞错 / 没有任何现场感 / 没有承接 也没人设印象目的 / 踩平台合规红线），建议直接重写。
- **合规红线一票否决**：成交型内容只要踩了上面的合规子项（违规导流、无资质做价格招揽、虚假夸大），无论其它项多好，直接判不通过——先改合规，再谈别的。
- **AI 味一票否决**：通用质检第 2/3/4/5/6 条**多条命中且密度高**（满篇「不是 X 而是 Y」翻转、工整对仗、零第一手细节、「太好了 / 以下是为您生成」式开头）→ 判定「**AI 味特别严重 = 不合格**」，无论其它项多好直接不通过，先改写成人话再过。**不分对象**——给谁写的稿子都一视同仁。

不要在不通过的情况下放水。

## 输出格式

```
## 内容类型
（流量 / 信任 / 成交 / 人设 / 价值观 / 思考）

## 通用质检
1. 空话：✅ / ❌
2. 模板腔：✅ / ❌
3. AI 腔：✅ / ❌
4. 机械排比：✅ / ❌
5. 重复句式：✅ / ❌
6. 书面词：✅ / ❌
7. 目标用户：✅ / ❌
8. 现场感：✅ / ❌
9. 结尾：✅ / ❌

## 分类型质检
（对应类型的清单逐条标）

## 问题清单

### 问题 1：____
问题句：> "____"
为什么是问题：____
改写方向：
- ____
- ____

### 问题 2：____
（同上）

## 判定
通过 / 改后通过 / 不通过
说明：____（如改后通过，列出要改的几处；如不通过，建议重写并指出重点）
```

## 质检标准（元质检：质检自己有没有做好）

- 内容类型有标记，质检按类型走。
- 通用质检 9 条逐条标过，没有漏检。
- 分类型质检按当前类型的清单走，没有把"流量型"的标准强加到"人设型"上。
- 每个问题摘了原稿句子，定位准确。
- 每个问题给了改写示例，不是只指方向。
- 判定结论明确（通过 / 改后通过 / 不通过），没有"基本通过""勉强通过"这种含糊话。
- 人设 / 价值观 / 思考型没有被错误地按"商业承接缺失"否决。
- 成交型没有被错误地按"人设印象弱"否决。
- 成交型（含价格 / 收费 / 召集 / 导流）有没有过合规红线子项——真查了平台官方规则原文，没过的有没有一票否决，而不是跳过。
