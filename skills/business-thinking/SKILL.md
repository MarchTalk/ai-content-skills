# business-thinking

## 什么时候调用

- 内容任务进入"为什么用户付费"环节。
- 内容数据好但不出单，需要从流量逻辑切换到成交逻辑。
- 用户提到价格、定价、利润、付费阻力、咨询拖延。
- 选题阶段判断一个角度有没有商业落点。
- 诊断阶段判断账号是不是只在博流量、漏掉了成交。

## 不要做什么

- 不要用学院派术语（赋能、闭环、势能、降维打击、底层逻辑、价值链、护城河、生态位）。
- 不要把商业拆解写成漂亮报告或咨询 PPT。
- 不要假设账号主已经懂自己的产品价值，多数人讲不清楚自己卖的是什么。
- 不要把"流量内容"和"成交内容"混着算。一条爆款不等于一笔订单。
- 不要用"提升转化率"这种没有具体动作的结论收尾。

## 工作流程

第一步：真伪需求

- 这个产品 / 服务有人为它真金白银付过钱吗？
- 付的是谁？付了多少？付的频率？复购了吗？
- 如果没有真实付费数据，把这一步标红，后面的所有结论都要打折。

第二步：翻译产品

- 用一句"用户能听懂"的话讲清楚这个产品在卖什么。
- 用户买的是结果、过程、承诺、还是身份感？
- 同一个产品对不同人群的"卖点"不一样，先锁人群再说卖点。

第三步：阻力盘点

用户不下单时，怕的是什么？必须落到下列六项之一：

- **钱**：买不起 / 觉得贵 / 担心买完不值。
- **风险**：买完不好用、不安全、出问题没人管、损失大于收益。
- **时间**：等不起、用不熟、学习成本太高。
- **信任**：不认识、没口碑、之前被坑过、看起来像骗子。
- **面子**：买完丢人、说不出口、家人不同意。
- **决策成本**：比来比去太累、决策链条太长、需要老婆 / 老板 / 合伙人拍板。

每条阻力要给具体场景（用户什么时候、在哪里、对谁、说什么话表达这种阻力）。

第四步：内容定位

判断当下这条内容的角色：

- **流量内容**：让陌生人刷到 / 记住 / 关注。
- **信任内容**：让看过的人觉得"这家懂行 / 不忽悠"。
- **成交内容**：让信任过的人推一把就下单。

三类内容写法不同，不能用同一套模板。

第五步：商业落点

每条结论必须翻译成具体动作。不要写"提升信任"，要写"评论区固定回复一条价格说明"。不要写"加强转化"，要写"主页第一条钉一支 3 分钟的成交答疑稿"。

## 输出格式

```
## 真假需求
真实付费证据：（订单 / 案例 / 价格区间 / 复购频率）
判定：真 / 假 / 数据不足

## 产品翻译
用户能听懂的一句话：「____」
卖点对应的人群：「____」

## 最大购买阻力
（钱 / 风险 / 时间 / 信任 / 面子 / 决策成本）中具体哪一项
场景：用户在 ____ 情况下，对 ____ 说 ____

## 这条内容的角色
流量 / 信任 / 成交

## 商业落点（具体动作）
- 动作 1：____
- 动作 2：____
- 动作 3：____
```

## 质检标准

- 落点是钱、风险、时间、信任、结果、代价中的具体一项，没有"价值""势能""赋能"这种抽象词。
- 区分了流量内容和成交内容，没有混淆。
- 没有用咨询术语和学院派词汇。
- 用户视角而不是老板视角（描述用户怎么想、怎么犹豫，而不是描述老板有多想卖）。
- 每条结论可以对应到一个明天就能做的具体动作。
- 真伪需求那一步如果没有付费证据，结论已经主动打折。
