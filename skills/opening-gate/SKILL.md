# opening-gate · 开头闸门

## 一个实测事实（这道闸门为什么单独存在）

一个图文账号 62 条笔记的后台数据：封面点击率 10% 以上的条全部升池（几千曝光起步），8% 以下的全部卡在首池几百曝光。同一个账号、同一个作者、同等的内容水平，胜负全在封面那一句话。先有点击，内容才被看见——点击率不过关，正文写成什么样都等于没写。

还有一条更扎心的对照：一条选题判定满分的笔记，封面句写成了中性提问，点击率 3.4%，直接沉底。**选题过了不等于封面句过了**，两层要分开过检。

## 什么时候调用

- 任何稿子成稿后、发布前，过封面句 / 标题 / 开头这一道。
- 用户问「开头怎么写」「标题起什么」「帮我看看这个封面句」。
- 已发布内容点击率低（发布 48 小时后低于 8%），回来改封面句重测。

## 不要做什么

- 不要跳过正文直接优化开头。开头憋不出来，九成原因是正文没有能亮的牌——没判断、没数据、没反差，先回去补正文，再回来写开头。
- 不要给中性提问放行。「X 到底有没有用？」这种没立场、没损失感的问句，观众划过去零成本，毙。
- 不要让行业概念当封面句主语。概念对概念（「自然流决定下限，付费流决定上限」）像教材金句，没人认领，毙。
- 不要编数字。「十个有九个」这类伪统计全文任何位置禁用；有真数据用真数据并给出处。
- 不要把开头写成引子。「有个客户跟我说……」这类铺垫式开口，前两秒就把人送走了。

## 工作流程

### 第一步：认平台和形态

| 形态 | 主战场 |
| --- | --- |
| 图文（小红书类） | 封面首行那句大字（注意：封面句和发布标题是两句话，分开写、分开检） |
| 口播短视频 | 前五秒的第一句 |
| 长文 / 公众号 | 标题 + 首段第一行 |

### 第二步：封面句 / 标题过三闸门

1. **句里有没有「人」**——普通人、老板、大多数人、你……观众要能在这句话里看到自己，或者看到自己讨厌的人。主语是行业概念 = 黑板报，毙。
2. **牌亮了没有**——判断句要把判决当场下完，狠话可以口语粗粝；疑问句必须自带立场或反常识答案（「能不能保证效果？肯定不能」）。注意分工：**亮的是判决，不是论证**——判决下完，观众点进来看的是「为什么」。中性提问、「聊聊两者的关系」、温和纠错 = 没亮牌，毙。
3. **说的是不是人话**——术语一律换普世词：商业闭环 → 赚到钱；自然流付费流 → 不花钱的流量 / 花钱的流量。能借当下热词热点就借。

实测正负样本对照（同一账号、同一封面格式，变量只有那句话）：

| 点击率 | 封面句 | 为什么 |
| --- | --- | --- |
| 18.9% | 从黄金白银事件，就能看出大多数人都是韭菜。 | 热点 + 狠判决 + 句里有人 |
| 14.9% | 互关互赞，这玩意除了自我感动，屁用没有 | 口语狠话，判决下完 |
| 10.1% | 代运营能不能保证效果？答案是：肯定不能的 | 问句但当场亮牌，答案反常识 |
| 5.0% | 养号，卖货？两者之间的关系 | 概念对概念，没有人 |
| 4.2% | 普通人如何通过自媒体，实现个人价值0-1商业闭环 | 「0-1商业闭环」是同行词，普通人看不懂 |
| 3.4% | 老板刷运营干货，到底有没有用？ | 选题满分但封面句不亮牌——中性疑问，没立场没损失感 |

### 第三步：开头五秒过三条

1. **独立成立**——不能假设观众看了封面和标题，第一句自带完整话题信息。
2. **零铺垫，直给**——第一句就是判断、事实、数字或原话直接砸。禁「有个客户跟我说」式引子，禁自问自答开场，禁报幕（「今天聊聊 X」）。
3. **判决和论证分工**——判决当场下，论证和机制留给正文。开头把「为什么」也讲完，观众就没有留下来的理由了。

### 第四步：素材硬度回检

开头亮的牌，正文接不接得住？开头下了判决，正文有没有机制和证据；开头给了数字，正文有没有出处。接不住 = 回去改正文，或者换一张小一点的牌，不许拿标题党硬撑——撑一次，掉的是账号的信用。

### 第五步：产出

- 封面句 / 标题：给 3-5 条候选，每条标触发器方向 + 一句为什么。实测验证点击率产出最高的四类：**争议挑衅、身份代入、认知冲突**（观点内容），**好奇缺口**（搜索型选题专用，配平台热词）。
- 开头：给 1-2 版改法，逐处引用原文说明改了什么。
- 最后一句话决断：推荐哪条、为什么，不模棱两可。

### 第六步：数据回流（发布之后这道闸门还没关）

发布 48 小时后看后台点击率：
- 10% 以上：升池达标。
- 8%-10%：观察，下一条同类选题优先打磨封面句。
- 8% 以下：先改封面句重发测试，**别先怀疑选题**。封面句改过两版还上不来，再回选题层找问题。
